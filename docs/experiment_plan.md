# 实验计划

## 阶段一：数据链路验证

1. 扫描 D2-City 文件并生成 `raw_index.json`。
2. 抽取关键帧并生成 `frame_index.json`。
3. 构造弱标注 SFT 样本。
4. 抽样检查图像路径、上下文和中文答案格式。

## 阶段二：LoRA 可行性实验

1. 使用 Qwen2-VL 或 Qwen2.5-VL Instruct 模型作为 base model。
2. 在弱标注 SFT 数据上进行 LoRA 微调。
3. 对 test split 运行 base 与 LoRA 对比。
4. 生成 Markdown 表格用于人工检查。

## 阶段三：论文实验扩展

1. 将弱标注升级为人工复核或半自动高置信标签。
2. 加入真实视觉事件检测输出，例如拥堵、事故、施工占道、异常停车。
3. 设计评价指标：风险等级准确率、异常事件 macro-F1、解释结构完整率、管理建议可用性评分。
4. 对比不同上下文设置：仅图像、图像加天气、图像加完整结构化信息。
5. 对比不同模型和训练策略：base、LoRA、不同 LoRA rank、不同数据规模。
